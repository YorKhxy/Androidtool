# ADR 0002: Pico 弱网控制采用助手 APK 与 TCP/UDP 覆盖
- 状态: Accepted
- 日期: 2026-05-29

## 背景

桌面工具需要控制 Pico 设备内某个第三方 APP 的网络质量，用于验证目标 APP 在弱网下的表现。核心诉求不是复刻完整 clumsy，而是提供可控的丢包、延迟、上行限速、下行限速。

约束很明确：

- 只能影响目标 APP，不能影响 Pico 整机网络
- 不能影响桌面工具与 Pico 的 WiFi ADB 连接
- 控制目标是任意第三方 APP，不能要求目标 APP 改代码
- Pico / VR 场景可能使用 UDP，因此只支持 TCP 会导致验收失真

## 决策

弱网控制采用“桌面工具 + Pico 弱网助手 APK”的双端架构：

1. 桌面工具
   - 负责选择设备、目标包名、弱网参数
   - 负责安装/启动/停止 Pico 弱网助手
   - 负责展示助手状态与错误原因

2. Pico 弱网助手 APK
   - 使用 Android `VpnService` 创建 VPN 通道
   - 使用按包名分流能力，只让目标 APP 进入弱网通道
   - 对目标 APP 的 TCP 与 UDP 流量施加丢包、延迟、上行限速、下行限速

3. 第一版协议覆盖
   - TCP 必须支持
   - UDP 必须支持
   - 不做完整 clumsy 功能，例如乱序、重复包、DNS 篡改、协议规则脚本、整机代理模式

## 后果

正向影响：
- 能满足“只限制某个 APP，不影响 ADB/WiFi 链路”的核心场景
- TCP 与 UDP 都纳入第一版，避免 VR/实时类应用测试失真
- 桌面工具仍保持控制台定位，不把网络转发复杂度塞进 Electron 主进程

代价：
- 需要新增 Android 助手 APK 子工程
- 需要处理 `VpnService` 授权、前台服务、目标包名分流、助手状态同步
- 弱网内核不能只靠简单 HTTP 代理，需要评估可维护的 TUN 转发方案

## 执行规则

- 第一版验收必须覆盖 TCP 与 UDP 两类流量
- 弱网参数最小集合为：丢包、延迟、上行限速、下行限速
- 弱网必须按目标包名生效，禁止默认影响整机网络
- WiFi ADB 链路必须保持可用，不能被弱网配置接管
- 若底层方案暂时无法可靠支持 UDP，必须明确标记为阻塞，不能偷偷发布为第一版完成
