# Product Spec Changelog

## 2026-05-28

- 性能快照：截图采集拆为 provider 链路，优先使用 raw framebuffer 快路径，失败时自动回退 PNG screencap，为后续实时流 / SDK 截图通道保留接入点。
- 性能采集：Android 与 Pico 性能页签不再采集或展示网络速度，网络抓包能力保留在网络页签。
- 性能导出：Raw Data 去掉 NET 字段，合并 Pico Metrics 数据，GPU % 排在 MEM MB 后面，不再单独生成 Pico Metrics 页签。

- 性能快照：补充单击快照缩略图打开大图预览的回看能力，支持遮罩、关闭按钮和 Esc 关闭预览。

## 2026-05-29

- 性能录制：新增短时性能录制需求，普通 Android 与 Pico 通过 provider 分流；第一版限制 10 / 30 / 60 秒，保存 MP4 与指标 manifest，不做无限通用录屏。
- 性能页 UI：新增取证操作区要求，录制与快照控制合并排布，避免性能页按钮继续挤占指标展示空间。
- 性能录制回看：工具内展示录制缩略图，单击缩略图播放视频；视频播放时按当前时间匹配性能采样，并把指标叠加在画面中。
- 性能录制路径：录制视频和 manifest 对渲染层只暴露 `performance-recordings/...` 相对路径，通过应用内媒体协议读取，避免依赖本机绝对路径。
- Pico 录制展示：Pico 录制缩略图和播放器按性能快照同样的单眼裁切显示，不展示双目并排画面；性能页记录区调整为快照在前、录制在后。
- 性能录制修正：最终 MP4 保留设备端原始视频，不再烧录性能指标水印；性能指标仅保存在 manifest，并在录制卡片与工具内播放器中按时间线动态展示。录制完成后仍清理设备端录屏进程并短暂等待，避免紧接着抓快照失败。

## 2026-05-30

### 新增
- 新增「投屏镜像与设备操控」模块（功能需求 2.5）：第一版采用路线 A——工具内打包 scrcpy 二进制，一键调起其原生窗口实现高帧率低延迟镜像，关闭窗口即停止；与性能页基于截图流的低帧率「实时预览」并存互补。
- 新增反向操控能力：触屏点击/滑动、电脑键盘文字输入、物理键（返回/Home/最近任务/电源/音量）经 scrcpy 原生快捷键触发；工具内提供快捷键速查表。
- 新增 Pico 单眼镜像：Pico/XR 设备启动 scrcpy 时通过 `--crop` 裁出单眼区域显示，操控按单眼坐标映射，与性能快照/录制单眼口径一致。
- 新增数据模型 `MirrorSession`（投屏会话配置：设备、裁切、分辨率、码率、状态、进程生命周期）；新增用户流程 4.4 投屏操控流程。
- 技术方案补充：主进程以子进程 `spawn` scrcpy，传入设备序列号与启动参数，负责参数拼装与进程生命周期回收；第一版不做内嵌解码（路线 B WebCodecs 内嵌镜像留作后续升级）。

### 能力边界说明
- 普通 Android 设备：镜像 + 触屏/文字/物理键操控完整可用。
- Pico 2D 界面：镜像 + 触屏点击可操作。
- Pico VR 沉浸场景 6DoF 手柄：**第一版明确不支持**（VR 手柄输入走 VR runtime，非标准 Android input 事件，scrcpy 无法注入）；HID 转发方案风险高，留作后续研究。

### 新增
- 新增「卸载应用」功能（功能需求 2.2）：经确认后走 `adb uninstall <包名>` 卸载，绕开投屏/VR 界面手动操作；手机与 Pico 通用。动机：Pico VR 界面靠手柄射线操作，投屏注入触摸无法可靠驱动应用管理；手机长按弹出菜单在投屏中也易误关。adb 卸载是设备类型无关的可靠路径。
- 新增「已安装应用列表」（功能需求 2.2）：设备页签展示设备所有第三方已安装应用（`pm list packages -3`），支持搜索过滤，每个应用可直接卸载。

### 修改
- 卸载入口调整：从进程页签移至设备页签。原因：进程列表只含运行中的进程且以系统进程为主，无法稳定呈现可卸载的应用；改用「已安装应用列表」作为正确数据源，覆盖未运行的已安装应用。

### 删除
- 移除「进程」与「Activity 栈」两个页签及其 UI（用户不再需要）。仅移除前端展示入口，主进程对应 IPC 能力暂保留。

### 新增（应用操作）
- 已安装应用列表新增「启动应用」（`monkey -p <包名> -c android.intent.category.LAUNCHER 1`）与「关闭应用」（`am force-stop <包名>`）两项操作，每个应用一行三按钮：启动 / 关闭 / 卸载。手机与 Pico 通用。
- 新增「批量安装」：单个 APK 并行安装到多台已连接设备。并发限流（默认 4，可调 2/4/8/不限），单台故障隔离 + 单独重试，逐台状态展示；安装模式支持保留数据重装（-r）与允许降级覆盖（-r -d）。复用单台 installApk 的 --no-streaming 兜底逻辑。说明：真正的「卸载后清数据全新安装」需读取 APK 包名，留作后续扩展。

### 修改（设备页精简）
- 移除设备页主区域的「设备详情」卡片（厂商/型号/系统/API 等信息在左侧「设备信息」区已有，重复）。
- 「息屏」「重启」从原卡片的「更多」菜单移至左侧设备卡，每台设备行内与「断开」并列，可按设备操作。
- 去除原卡片的「开始日志」按钮（日志页签已提供启停入口）。

### 修改（安装 UI 整合）
- 单设备安装与批量安装整合为**同一个「应用安装」面板**：选 1 个或多个 APK + 勾选 1 台或多台设备（默认当前设备）+ 并发上限，一键并行安装。**每台设备展示各自的安装队列与进度条**（含整体百分比、逐 APK 进度、状态、错误、删除、按设备重试失败）。移除独立的 BatchInstallPanel 组件，统一复用 per-device 队列 + 进度计时器基础设施。

## 2026-06-01

### 新增
- 新增「历史设备保存与快速重连」能力（功能需求 2.1，新增 2.1.1）：仅保存通过 WiFi 成功连过的设备（USB 即插即识别不入历史），WiFi 连接成功后自动写入历史，无需手动收藏。
- 新增「历史设备卡片」列表（设备连接区域）：按最近连接时间倒序排列，每张卡片展示设备型号、SN、上次 IP:端口、上次连接时间、在线/离线状态，提供「快速连接」按钮一键用记录的 IP:端口发起 adb connect。
- 新增失败重连交互：快速连接失败（设备 IP 变更）时，卡片就地展开 IP 输入框（预填上次 IP），用户改完后确认重连；成功则用新 IP 覆盖历史记录，仍失败则保留输入框与历史记录供再试。
- 新增「移除历史」：每张历史卡片提供移除入口，带二次确认，仅删除历史记忆不影响当前已建立的连接。
- 新增数据模型 `HistoryDevice`（数据模型 5.1.1）：serial（SN，唯一标识去重）、model、lastAddress（IP:端口）、lastConnectedAt（排序）、online（运行时态不持久化）；持久化到用户数据目录（app.getPath('userData')），UI 不暴露宿主绝对路径。

### 修改
- 用户流程 4.1 连接设备流程：连接方式新增「历史设备快速重连」分支，补充 WiFi 首次连接成功自动写入历史、历史快速连接失败时就地输入新 IP 重连两条路径。
