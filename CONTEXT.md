# 领域词汇

## FPS
- 在本项目中，`FPS` 默认指“前台应用渲染 FPS”。
- 只有在明确标注为 `Pico FPS` 时，才允许它表示 PICO Metrics 中的头显实时帧率。
- 除非明确说明，否则不把设备屏幕刷新率（60Hz / 90Hz / 120Hz）当作同义词。

## 实时预览设备画面
- 指在性能模块内持续查看当前设备画面，用于和性能指标联动排查问题。
- 当前语义偏“性能诊断中的实时预览”，不是独立的远程控制模块。

## 性能快照
- 指在某一时间点同时记录性能指标与设备截图的取证对象。
- 最小信息集合应包括：时间戳、设备 ID、当前性能指标、前台应用或 Activity、截图。
- 当设备是 Pico 时，快照中的性能指标必须优先保留 PICO Metrics 原始口径。

## 性能录制
- 指性能模块内的短时取证录制，不是通用长时间录屏功能。
- 最小信息集合应包括：设备画面 MP4、起止时间、设备 ID、录制 provider、前台应用或 Activity、录制期间性能采样序列。
- 普通 Android 与 Pico 必须走 provider 分流；Pico 第一版可复用设备端录制通道，但语义上保留 `pico-sdk` 或实时流接入空间。
- 录制视频和 manifest 在工具内使用 `performance-recordings/...` 相对路径定位，由应用内媒体协议映射到运行目录，不能把本机绝对路径作为 UI 查找依据。
- 最终 MP4 文件必须保留设备端录制下来的原始画面，不把性能指标烧录或水印进视频文件。
- 性能指标只存入 manifest 的采样时间线；工具录制卡片和工具内播放器按视频时间动态叠加展示指标。

## 性能诊断工作流
- 指“观察性能指标 -> 联动查看设备画面 -> 触发截图取证 -> 回看快照定位问题”的完整排障路径。

## Pico 性能视图
- 指当连接设备被识别为 Pico 时，性能页签切换为 PICO Metrics 口径与布局。
- `Pico 性能视图` 不复用 Android 通用性能卡片语义，不把 `CPU 使用率 / 内存使用率 / 网络速度` 当作主指标。
- 第一版 Pico 性能视图默认围绕 `FPS / MTP / FrmCpu / FrmGpu / ATWGPU / GPU` 展示。

## Pico 官方指标支持范围
- 指 `Pico 性能视图` 中“官方口径指标”可被视为有效的前提边界。
- 当前项目只承诺支持“已集成 `XR Profiling Toolkit` 的 Pico 应用”。
- 未集成 `XR Profiling Toolkit` 的任意第三方 Pico 应用，不属于官方指标能力范围；即使设备已连接，也不能把缺失指标视为产品 Bug。

## 性能数据 Provider
- 指性能模块按设备类型切换的数据采集与展示策略。
- 当前项目约定至少存在两类 Provider：`android`（通用 Android）与 `pico`（PICO Metrics）。
